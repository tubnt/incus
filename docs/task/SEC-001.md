## 描述

实现 PLAN-019 定义的五项安全/审计基线能力：

1. **Step-up auth**：敏感操作（删 VM / 迁移 / 踢节点 / 充值 / 密码重置）走 Logto OIDC 重认证（`prompt=login&max_age=0`），回调读 `auth_time` claim 校验 5 分钟内已重认证
2. **全量审计打点**：middleware 拦截所有管理员 POST/PUT/PATCH/DELETE，handler 层补业务 target + details；敏感字段 redact；CI 脚本保证覆盖率 ≥ 90%
3. **审计导出 + 保留策略**：CSV 导出（带筛选）+ koanf 配置 `audit_retention_days`（默认 365）+ 每日清理 worker
4. **API Token TTL + 续签**：默认 24h、支持 1h-90d 自定义；续签接口；前端剩余时间展示 + 续签按钮；过期 token 清理
5. **Shadow Login**：独立 JWT session（30min，actor_id + target_user_id + reason）；金钱类路由 403；顶栏红横幅；审计标注

### 验收标准

- [ ] 敏感路由触发 step-up redirect，auth_time 过期返 401 {error:"step_up_required"}
- [ ] admin 写操作审计覆盖率脚本报告 ≥ 90%
- [ ] 导出 CSV 日期范围/操作类型筛选生效，敏感字段 redact
- [ ] API Token 创建可指定 TTL，续签后旧 token 立即失效
- [ ] Shadow Login 下发起充值接口返 403，顶栏红横幅显示剩余时间
- [ ] `go test ./... && bun run typecheck && bun run build` 全过
- [ ] Staging E2E 验证 5 项流程（见 PLAN-019 Phase F）
